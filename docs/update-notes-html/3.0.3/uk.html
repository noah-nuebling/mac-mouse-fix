Усунення збоїв при прокручуванні

Починаючи з Mac Mouse Fix 3.0.2 надходили численні повідомлення про те, що Mac Mouse Fix періодично вимикається та знову вмикається під час прокручування. Це було спричинено збоями фонового застосунку ‘Mac Mouse Fix Helper’. Це оновлення намагається виправити ці збої за допомогою наступних змін:

  • Механізм прокручування спробує відновитися та продовжити роботу замість того, щоб аварійно завершитися, коли він зустрічає крайній випадок, який, схоже, призводив до цих збоїв.

  • Я змінив спосіб обробки неочікуваних станів у застосунку загалом: замість того, щоб завжди негайно аварійно завершуватися, застосунок тепер у багатьох випадках спробує відновитися від неочікуваних станів.

    • Ця зміна сприяє виправленням збоїв при прокручуванні, описаних вище. Вона також може запобігти іншим збоям.

Примітка: я ніколи не міг відтворити ці збої на своєму комп’ютері, і я досі не впевнений, що їх спричинило, але на основі отриманих повідомлень це оновлення має запобігти будь-яким збоям. Якщо ти все ще стикаєшся зі збоями під час прокручування або якщо ти стикався зі збоями у версії 3.0.2, буде цінно, якщо ти поділишся своїм досвідом та діагностичними даними в GitHub Issue #988. Це допоможе мені зрозуміти проблему та покращити Mac Mouse Fix. Дякую!

Усунення затримок при прокручуванні

У версії 3.0.2 я вніс зміни до того, як Mac Mouse Fix надсилає події прокручування до системи, намагаючись зменшити затримки при прокручуванні, ймовірно спричинені проблемами з API VSync від Apple.

Однак після більш ретельного тестування та зворотного зв’язку виявилося, що новий механізм у 3.0.2 робить прокручування плавнішим у деяких сценаріях, але більш уривчастим в інших. Особливо у Firefox воно здавалося помітно гіршим.
Загалом, не було очевидно, що новий механізм дійсно покращив затримки при прокручуванні в цілому. Крім того, він міг сприяти збоям при прокручуванні, описаним вище.

Тому я вимкнув новий механізм і повернув механізм VSync для подій прокручування до того, яким він був у Mac Mouse Fix 3.0.0 та 3.0.1.

Дивись GitHub Issue #875 для додаткової інформації.

Повернення коштів

Мені шкода за незручності, пов’язані зі змінами прокручування у версіях 3.0.1 та 3.0.2. Я значно недооцінив проблеми, які з цим виникнуть, і повільно вирішував ці питання. Я зроблю все можливе, щоб навчитися на цьому досвіді та бути обережнішим з такими змінами в майбутньому. Я також хотів би запропонувати повернення коштів усім, кого це торкнулося. Просто натисни тут, якщо тебе це цікавить.

Розумніший механізм оновлень

Ці зміни були перенесені з Mac Mouse Fix 2.2.4 та 2.2.5. Ознайомся з їхніми примітками до випуску, щоб дізнатися більше про деталі. Ось короткий виклад:

  • Є новий, розумніший механізм, який вирішує, яке оновлення показати користувачеві.
  • Перехід від використання фреймворку оновлень Sparkle 1.26.0 до останньої версії Sparkle 1.27.3.
  • Вікно, яке застосунок відображає, щоб повідомити тебе про доступність нової версії Mac Mouse Fix, тепер підтримує JavaScript, що дозволяє краще форматувати примітки до оновлення.
